# Contributing

Thank you for contributing to the FabricOps Starter Kit.

## Contribution guidelines

- Keep examples synthetic.
- Keep the framework Fabric-friendly but not tenant-specific.
- Prefer simple Python-first APIs.
- Document before implementing complex features.
- Include tests for future logic.

## Scope reminder

This repository currently emphasizes documentation and scaffolding. Core engines (profiling, drift, quality, contracts) should be introduced in staged PRs.
