# Framework Modules (Conceptual)

Concept-only module definitions are documented here so Python package code remains under `src/fabric_data_product_framework/`.

## profiling
Generate source/output profiles for metadata drafting, baseline statistics, and drift thresholds.

## rule_generation
Convert metadata and requirements into suggested DQ rules for human review.

## validation
Execute approved data quality and contract checks in notebook/pipeline runs.

## drift_detection
Detect schema and data drift and publish outcomes to central logs.

## governance_labeling
Suggest and apply governance/sensitivity labels with human approval checkpoints.

## documentation_generation
Generate runbooks, test plans, and handover packs from approved metadata and validation logs.
