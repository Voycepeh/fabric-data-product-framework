# Onboarding Playbook for Junior Engineers

This playbook complements the lifecycle model and notebook split.

- Start with [Lifecycle operating model](../lifecycle-operating-model.md)
- Use [Notebook structure](../notebook-structure.md)
- Apply accountability boundaries from [AI in the loop](../ai-in-the-loop.md)

## Practical onboarding sequence

1. Capture purpose, steward, and approved usage.
2. Fill business metadata and governance labels.
3. Draft source declarations and contract expectations.
4. Run profiling and metadata logging.
5. Complete separate EDA notes.
6. Implement transformations and run checks/gates.
7. Export handover package and AI context.
