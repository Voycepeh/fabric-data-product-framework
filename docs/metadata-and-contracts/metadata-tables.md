# Metadata tables

This page describes the metadata-backed source of truth used to assemble the FabricOps handover contract output.

## Canonical metadata/contract store tables

- `contracts`
- `contract_columns`
- `contract_rules`
- `quality_results`
- `lineage_records`

## Role-aligned ownership

| Table | Primary ownership | Purpose |
| --- | --- | --- |
| `contracts` | Governance steward | Agreement scope, approved usage, business context, ownership, restrictions, classification, sensitivity/PII context. |
| `contract_columns` | Governance steward + analyst/data scientist | Column-level metadata, semantic interpretation, approved classification/sensitivity context. |
| `contract_rules` | Analyst / data scientist (validated), enforced by data engineer | Approved DQ and contract rules for deterministic enforcement. |
| `quality_results` | Data engineer | Pipeline execution quality outcomes and quarantine evidence. |
| `lineage_records` | Data engineer | Source-to-target lineage and transformation evidence per execution context. |

## Assembly and export

Handover/data contract outputs are generated by assembling approved metadata and evidence from these tables.

The assembled output is designed for Open Data Contract-compatible YAML/JSON export.

## Core operational loop

Step 5 → Step 2: execution evidence in `quality_results` and `lineage_records` feeds governance and analyst updates to approved metadata for future runs.
